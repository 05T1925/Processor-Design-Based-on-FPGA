# 风险点与降级方案

用途：提前定义项目风险、触发条件和降级路径，避免一周项目失控。

最后更新时间：2026-07-07

## 1. 风险等级

- 高：可能阻塞上板或验收。
- 中：影响亮点但不影响保底。
- 低：影响展示效果或报告材料。

## 2. 风险表

| 风险 | 等级 | 表现 | 降级方案 |
|---|---|---|---|
| MAC 第三读口复杂 | 中 | regfile 接口和写回冲突 | 降级为 x31 累加 MAC |
| 100MHz timing 不过 | 高 | negative slack | MAC 改多周期，报告解释性能/时序权衡 |
| 上板数码管异常 | 中 | 显示乱码/不亮 | 先用 LED 显示低 8 位结果 |
| BRAM 初始化麻烦 | 高 | 仿真可跑，上板程序不对 | 先仿真用 `$readmemh`，上板再确认 Vivado 初始化流程 |
| 流水线来不及 | 中 | hazard 调不完 | 保留为冲刺文档和展望，不影响主线 |
| `.xdc` 不完整 | 高 | 无法正确绑定引脚 | 以已核对的 `constraints/minisys.xdc` 为主线，缺少资源暂不进入第一版 |
| 误用其他板卡约束 | 高 | 引脚和电平不匹配，可能无法上板 | 禁止使用 Nexys4DDR/Basys3/EGO1/TEC-PLUS 约束；以 `docs/design/board_constraints_audit.md` 为准 |
| 数码管有效电平搞反 | 中 | 显示乱码、全亮或全灭 | Minisys 共阳极，`seg/an` 低电平有效；先用单数字扫描测试 |
| 顶层端口名和 `.xdc` 不一致 | 高 | Vivado 报未约束端口或约束找不到端口 | `minisys_top` 端口必须和 `constraints/minisys.xdc` 一致，内部命名只在映射层转换 |
| 机器码手写出错 | 中 | CPU 看似 bug，实为程序错 | 手工对照编码表；第二阶段写 `gen_mem.py` |
| AI 生成代码接口不一致 | 中 | 模块难集成 | 先更新 interfaces.md，单模块生成，人工 review |
| Git 合并冲突 | 低 | 多人同时改公共文档/接口 | feature 分支，小步提交，组长合并 |

## 3. 必须遵守的降级口径

- MAC 第三读口复杂：降级为 x31 累加 MAC。
- 100MHz timing 不过：MAC 改多周期，报告解释性能/时序权衡。
- 上板数码管异常：先用 LED 显示低 8 位结果。
- BRAM 初始化麻烦：先仿真用 `$readmemh`，上板再确认 Vivado 初始化流程。
- 流水线来不及：保留为冲刺文档和展望，不影响主线。
- 约束风险：只使用 `constraints/minisys.xdc`，新增外设前先补审计记录。

## 4. 触发降级的建议时间点

- 第 2 天末：基础 CPU 仍不能跑 basic，立即缩减指令集。
- 第 3 天末：SoC 仍不能仿真 MMIO，先 LED-only 演示。
- 第 5 天末：MAC 仍不稳定，保留仿真证明，硬件演示回退基础 CPU。
- 第 6 天末：流水线未稳定，停止流水线开发，转报告和上板材料。
