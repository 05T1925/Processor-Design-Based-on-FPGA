## 一、CPI 公式与分量

### 1.1 流水线 CPI 通用公式

```
CPI_pipeline = CPI_ideal + CPI_load_use + CPI_jump + CPI_branch_mispred

其中:
  CPI_ideal          = 1.000  (每周期发射 1 条指令, 无冒险)
  CPI_load_use       = P(load) × P(use_after_load) × penalty(1)
  CPI_jump           = P(jump) × penalty(1)     [JAL + JALR]
  CPI_branch_mispred = P(branch) × P(mispred) × penalty(1)
```

### 1.2 指令混合比（RV32I 典型程序）

| 指令类型 | 占比 | 说明 |
|------|:---:|------|
| ALU (R/I-type) | 45% | ADD/SUB/AND/OR/XOR/SLL/SRL/SRA/SLT |
| Load (LW) | 20% | 内存读取 |
| Store (SW) | 10% | 内存写入 |
| Branch (条件分支) | 16% | BEQ/BNE/BLT/BGE/BLTU/BGEU |
| Jump (JAL/JALR) | 4% | 无条件跳转 |
| Others (LUI/AUIPC/MAC/EBREAK) | 5% | |

## 四、分支预测正确率与 CPI 关系

| BTB 正确率 | Branch CPI 贡献 | 总 CPI | vs. 静态加速 |
|:---:|:---:|:---:|:---:|
| 50% (静态) | +0.080 | 1.120 | 1.000× |
| 60% | +0.064 | 1.104 | +1.4% |
| 70% | +0.048 | 1.088 | +2.9% |
| 80% | +0.032 | 1.072 | +4.3% |
| **85% (保守预期)** | **+0.024** | **1.064** | **+5.0%** |
| **90% (乐观预期)** | **+0.016** | **1.056** | **+5.7%** |
| 95% | +0.008 | 1.048 | +6.4% |
| 100% (完美) | 0.000 | 1.040 | +7.1% |

> **关键洞察**：BTB 正确率从 50%→85% 即可获得大部分收益 (+5.0%)。
> 从 85%→95% 的额外收益仅 +0.7%，边际效益递减。

## 五、冒险惩罚按指令序列分析

### 5.1 典型 RAW 冒险序列（转发消除）

```
序列: ADD x1,x2,x3 → ADD x4,x1,x5
时序:
  C1: IF(ADD1)  ID(—)     EX(—)     MEM(—)    WB(—)
  C2: IF(ADD2)  ID(ADD1)  EX(—)     MEM(—)    WB(—)
  C3: IF(...)   ID(ADD2)  EX(ADD1)  MEM(—)    WB(—)
  C4: IF(...)   ID(...)   EX(ADD2)  MEM(—)    WB(ADD1)
                           ↑ FWD: EX/MEM→EX, 0 周期惩罚 ✓
```

### 5.2 Load-Use 冒险序列（需停顿）

```
序列: LW x1,0(x2) → ADD x4,x1,x5
时序:
  C1: IF(LW)   ID(—)    EX(—)    MEM(—)   WB(—)
  C2: IF(ADD)  ID(LW)   EX(—)    MEM(—)   WB(—)
  C3: IF(ADD)  ID(ADD)  EX(LW)   MEM(—)   WB(—)   ← PC 停顿
  C4: IF(...)  ID(NOP)  EX(ADD)  MEM(LW)  WB(—)   ← 气泡注入
                         ↑ STALL!              ↑ MEM/WB→EX 转发
  C5: IF(...)  ID(...)  EX(...)   MEM(ADD) WB(LW)
```

惩罚：**1 周期**。LW 数据在 MEM 阶段才可用，无法提前到 EX 转发。

### 5.3 分支误预测序列（BTB 正确预测 → 零惩罚）

```
序列 (BTB 正确预测 taken):
  C1: IF(BEQ)  [BTB lookup → HIT, counter=ST → predict TAKEN]
  C2: IF(TGT)  ID(BEQ)   ← PC 已被 BTB 重定向到 target!
  C3: IF(...)  ID(TGT)   EX(BEQ)  → branch taken, BTB was RIGHT → NO flush!
              ↑ 无气泡，流水线满载 ✓

序列 (BTB 误预测):
  C1: IF(BEQ)  [BTB lookup → MISS 或 counter=WNT → predict NOT-TAKEN]
  C2: IF(PC+4) ID(BEQ)   ← PC 取 PC+4 (预测不跳转)
  C3: IF(...)  ID(PC+4)  EX(BEQ)  → branch taken, BTB was WRONG → FLUSH!
  C4: IF(TGT)  ID(NOP)   EX(PC+4) → 气泡
              ↑ 误预测惩罚 = 1 周期
```

## 六、CPI 改善归因分析

```
CPI 改善来源          | 贡献 (绝对值) | 贡献 (百分比)
----------------------+-------------+-------------
转发 (RAW 消除)        |   +0.080     |  对多周期基线而言
Load-Use 停顿          |   -0.020     |  负面 (FSM 无此问题)
JAL/JALR 刷新          |   -0.020     |  负面 (FSM 无此问题)
流水线并行度 (5条指令)   |   +3.000     |  核心收益: CPI 4.0→1.0
BTB 分支预测改善       |   +0.088     |  BTB 相比静态的增量
----------------------+-------------+-------------
总计 (多周期→流水线+BTB) | +2.920     |  CPI 4.0→1.08
```
