## 一、MAC 自定义指令加速分析

### 1.1 点积运算实测数据（Icarus 验证通过 ✅）

| 指标 | 普通 RV32I | MAC 加速版 | 改善 |
|------|:---:|:---:|:---:|
| 测试输入 | A=[1,2,3,4], B=[5,6,7,8] | 同左 | |
| 期望结果 | 1×5+2×6+3×7+4×8 = 70 | 同左 | |
| 实际结果 | 70 ✅ | 70 ✅ | |
| 时钟周期 | 62 | 54 | **-12.90%** |
| 退休指令数 | 15 | 13 | **-13.33%** |
| CPI | 4.1333 | 4.1538 | +0.5% (可忽略) |
| MAC 指令执行次数 | 0 | 4 | |
| **加速比** | 1.0000 | **1.1481** | **+14.81%** |

### 1.2 加速比来源分解

```
加速比 = 62/54 = 1.1481

来源分析:
  指令数减少: 15→13 (减少 2 条)  → 直接贡献
  周期减少:   62→54 (减少 8 周期) → 2 条指令 × 4 周期/指令 = 8 周期

排除 EBREAK 停机开销 (各 2 周期):
  普通版: (62-2)周期 / 15条指令 = 4.0 CPI
  MAC版:  (54-2)周期 / 13条指令 = 4.0 CPI
  
  → 两版主体 CPI 完全一致 (4.0)
  → 加速 100% 来源于指令数减少
  → MAC 单条指令完成"乘+累加"两步操作
```

> 届时加速比将显著提升。

## 二、流水线加速分析

### 2.1 多周期 → 流水线 性能跃升

| 指标 | 多周期 FSM | 5级流水线+BTB | 提升 |
|------|:---:|:---:|:---:|
| CPI | 4.0 | ~1.08 | **-73%** |
| 吞吐量 @100MHz | ~25 MIPS | ~92 MIPS | **+268%** |
| IPC (每周期指令数) | 0.25 | ~0.93 | **+272%** |
| 最大并行指令数 | 1 | 5 | **5×** |
| 分支延迟 | 3 周期 | 1 周期 (仅误预测) | **-67%** |
| LW 延迟 | 5 周期 | 5 周期 (但隐藏 4 周期) | **有效 -80%** |

### 2.2 流水线阶段加速贡献

| 优化 | 贡献 | 实现方式 |
|------|:---:|------|
| 五级并行执行 | CPI 4.0 → 1.0 (理论) | IF/ID/EX/MEM/WB 重叠 |
| 转发 (forwarding) | 消除 ALU→ALU RAW 停顿 | 3 条旁路 (EX/MEM, MEM/WB×2) |
| Load-Use 停顿 | -0.02 CPI (不可避免) | 1 周期 stall + MEM/WB 转发 |
| 分支预测 (BTB) | CPI 1.16 → 1.08 | 16 条目 2-bit 饱和计数器 |
| 综合效果 | CPI 4.0 → 1.08 | **3.70× 加速** |

## 四、投入产出分析 (ROI)

### 4.1 各优化项的性价比

| 优化项 | 面积增量 (LUT+FF) | 性能增量 (MIPS) | ROI (MIPS / 100 LUT-FF) | 优先级 |
|------|:---:|:---:|:---:|:---:|
| **五级流水线** | +702 | +61 | **8.7** | ⭐⭐⭐⭐⭐ |
| 转发 (forwarding) | +400 | (含在流水线中) | — | ⭐⭐⭐⭐⭐ |
| **BTB 分支预测** | **+421** | **+6** | **1.4** | ⭐⭐⭐ |
| MAC 自定义指令 | +50 | +4 (点积) | 8.0 | ⭐⭐⭐⭐ |
| DSP48E1 流水化 | +0 | +0 (同 CPI) | — | ⭐⭐ |
| VEC_MAC_4 向量化 | +150 | +15 (矩阵) | 10.0 | ⭐⭐⭐⭐ (P2 冲刺) |

> **核心结论**：五级流水线是 ROI 最高的优化（+268% 性能 / +114% 面积 = 2.35× 效率）。
> BTB 的 ROI 相对较低（+7% 性能 / +17% 面积 = 0.42×），但其在分支密集程序中的价值显著。
